## Бизнес-цель
Цель этого проекта — разработка **масштабируемой рекомендательной системы (RecSys)** для стриминговой платформы (на примере датасета MovieLens). 

Модель формирует персонализированную выдачу Топ-K релевантных фильмов для каждого пользователя на основе истории его взаимодействий. Внедрение качественного алгоритма ранжирования напрямую влияет на ключевые продуктовые метрики: увеличивает **удержание пользователей (Retention Rate)**, **время смотрения (Watch Time)** и, как следствие, **LTV (Life-Time Value)** клиента.

## Ключевые инженерные и аналитические решения

В этом проекте я реализовала полный цикл (End-to-End) создания ML-продукта, применив индустриальные (Senior-level) практики проектирования рекомендательных систем:

* **Big Data ELT пайплайн (PySpark):** Для обработки данных использован Apache Spark. Агрегации признаков (user/item features) и расчет сложных метрик (любимый жанр пользователя через оконные функции `ROW_NUMBER() OVER`) реализованы на SQL-движке Spark, что позволяет легко масштабировать пайплайн на терабайты данных.
* **Генерация негативных примеров (Negative Sampling):** Пользователи оценивают только то, что смотрят (Positive Implicit Feedback). Чтобы модель научилась отличать релевантный контент от нерелевантного, был реализован алгоритм семплирования негативных примеров (фильмов, с которыми юзер не взаимодействовал) в соотношении 1:1.
* **Learning-to-Rank (LTR):** Вместо классической регрессии или бинарной классификации применена специализированная архитектура ранжирования. Модель `CatBoostRanker` обучалась с использованием функции потерь **YetiRank** — передового алгоритма оптимизации напрямую под метрики качества поиска.
* **Zero Data Leakage валидация:** Использован `GroupShuffleSplit` по `userId`. Это гарантирует, что история одного пользователя не разорвется между train и test, что предотвращает утечку данных (Data Leakage) и имитирует реальную работу системы с новыми (Cold Start) или существующими пользователями.
* **Микросервисная архитектура (FastAPI):** Инференс модели вынесен в отдельный класс и обернут в REST API с помощью FastAPI. Приложение готово к деплою в Docker и интеграции с Backend-частью продукта.

## Архитектура пайплайна

Пайплайн полностью реализован в едином Google Colab Notebook для удобства воспроизведения:
1. **Data Ingestion:** Автоматическая загрузка и распаковка датасета MovieLens.
2. **PySpark ELT:** Трансформация сырых логов (ratings.csv, movies.csv) в витрину фичей (Feature Store) в формате Parquet.
3. **Data Preparation:** Применение Negative Sampling и джоин фичей (User Features + Item Features).
4. **Model Training:** Обучение `CatBoostRanker` с оптимизацией под метрику `NDCG@10`.
5. **API Deployment:** Запуск фонового асинхронного сервера FastAPI прямо в среде Colab (через `nest_asyncio` и `threading`) для демонстрации работы эндпоинта `/recommend/{user_id}`.

## Результаты и бизнес-инсайты

* **Оптимизация метрик:** Модель успешно оптимизирует метрику **NDCG (Normalized Discounted Cumulative Gain)**, которая учитывает не только релевантность рекомендаций, но и их позицию в выдаче (чем выше релевантный фильм, тем лучше).
* **Анализ важности признаков (Feature Importance):** 
  * Признак *совпадения жанра (is_genre_match)* и *историческая средняя оценка пользователя* стали главными драйверами модели.
  * *Популярность фильма (item_total_views)* помогает модели рекомендовать уверенный "масс-маркет" пользователям с короткой историей (решение проблемы холодного старта).
* **Производительность:** Благодаря сохранению признаков в in-memory словари на этапе инференса, API отдает предсказание за миллисекунды, что критически важно для Highload-систем.

## Как запустить (В один клик!)

Проект специально настроен для мгновенного запуска в облаке. Вам не нужно скачивать датасеты или настраивать локальное окружение.

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](ССЫЛКА_НА_ТВОЙ_НОУТБУК_В_COLAB)

1. Нажмите на кнопку **Open in Colab** выше.
2. В верхнем меню выберите `Runtime` -> `Run All` (Среда выполнения -> Выполнить все).
3. Ноутбук автоматически скачает данные, поднимет Spark-сессию, обучит модель и запустит тестовые запросы к поднятому FastAPI серверу прямо в выводе последней ячейки!

### Локальный запуск
1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/valanchick/Enterprise-Grade-Movie-Recommender-System-PySpark-Learning-to-Rank.git
   ```
2. Установите зависимости:
   ```bash
   pip install pyspark catboost fastapi uvicorn nest-asyncio pandas numpy scikit-learn
   ```
3. Запустите Jupyter Notebook:
   ```bash
   jupyter notebook recsys_pipeline.ipynb
   ```
